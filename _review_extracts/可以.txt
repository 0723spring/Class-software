[Heading 1] 一、最终方案改动定版
[Heading 2] 1. 项目定位
[Normal (Web)] 项目名称不变：
[HTML Preformatted] 城市道路突发事件应急抢修仿真与辅助决策系统
[Normal (Web)] 最终做成一个轻量完整 Web 系统，支持：
[HTML Preformatted] 路网管理 事件录入 资源管理 路径规划 方案生成 仿真推演 评估报告 JSON 导入导出
[Normal (Web)] 系统主流程是：
[HTML Preformatted] 加载虚拟路网和抢修资源 ↓ 创建道路突发事件 ↓ 事故路段变为 damaged / closed ↓ 系统计算抢修队到事故点路径 ↓ 生成最近队伍优先、专业能力优先、多队伍协同三类方案 ↓ 按综合评分推荐最优方案 ↓ 启动仿真，更新队伍状态、道路状态和修复进度 ↓ 输出评估报告并支持 JSON 导出
[Heading 2] 2. 技术栈最终改动
[Normal (Web)] 原文档里写的是：
[HTML Preformatted] Vue3 / React + FastAPI / Spring Boot + JSON
[Normal (Web)] 最终实际实现改成：
[HTML Preformatted] 前端：原生 HTML + CSS + JavaScript + SVG 后端：Python FastAPI 数据：本地 JSON 文件 运行：python main.py 后访问系统页面
[Normal (Web)] 这个改动可以这样解释：
[Normal (Web)] 本系统采用逻辑前后端分离、部署集成运行的 B/S 架构。前端页面由 FastAPI 静态托管，前端仍通过 REST API 与后端交互。由于课程项目时间有限，最终前端采用原生 HTML/CSS/JavaScript + SVG 实现，以减少 Node/Vite 环境依赖，保证项目可快速运行和验收。
[Normal (Web)] 这个口径和概要设计不冲突，因为概要设计本身已经写了 FastAPI、JSON 文件和集成运行方式。

[HTML Preformatted] 主流程闭环 + M2/M3/M4 管理补齐 + JSON 导入导出 + 异常保护

[Heading 1] 四、数据结构标准
[Heading 2] 1. RoadNode 道路节点
[HTML Preformatted] { "id": "A", "name": "中心路口", "x": 120, "y": 180, "type": "intersection" }
[Normal (Web)] 字段要求：

[Heading 2] 2. RoadEdge 道路边
[HTML Preformatted] { "id": "AB", "fromNode": "A", "toNode": "B", "length": 1.2, "speed": 40, "flow": 900, "status": "normal" }
[Normal (Web)] 状态枚举：
[HTML Preformatted] normal 正常 congested 拥堵 damaged 损坏 closed 封闭 repairing 修复中 recovered 已恢复
[Normal (Web)] 校验规则：
[HTML Preformatted] id 唯一 fromNode 和 toNode 必须存在 length > 0 speed > 0 flow >= 0 status 必须是合法状态
[Heading 2] 3. EmergencyEvent 突发事件
[HTML Preformatted] { "id": "E001", "type": "road_collapse", "roadId": "HI", "severity": "high", "startTime": "08:10", "workload": 120, "blocked": true, "status": "created" }

[HTML Preformatted] road_collapse 道路塌陷 waterlogging 暴雨积水 bridge_damage 桥梁损坏 pipe_burst 管线爆裂 traffic_accident 普通事故占道

[HTML Preformatted] low medium high
[Normal (Web)] 事件状态：
[HTML Preformatted] created planned running finished cancelled
[Normal (Web)] 联动规则：

[Heading 1] 七、后端要做什么
[Normal (Web)] 后端负责数据、业务逻辑、算法、接口、校验。
[Heading 2] 后端文件建议
[HTML Preformatted] main.py storage.py algorithms.py services.py
[Normal (Web)] 赶时间可以合并，但逻辑要分函数。
[Heading 2] 1. 数据访问标准
[Normal (Web)] 必须封装：
[HTML Preformatted] load_json(filename, default) save_json(filename, data)
[Normal (Web)] 要求：
[HTML Preformatted] JSON 文件不存在时自动创建默认数据。 JSON 文件损坏时返回错误或默认数据，系统不崩溃。 导入失败不覆盖原数据。 写入 JSON 时使用 ensure_ascii=False, indent=2。
[Heading 2] 2. 业务逻辑标准
[Normal (Web)] 后端必须实现这些函数或同等逻辑：
[HTML Preformatted] create_event() update_event() delete_event() create_node() update_node() delete_node() validate_network() create_edge() update_edge() delete_edge() create_team() update_team() delete_team() create_depot() update_depot() delete_depot() find_shortest_path() generate_plans() calculate_score() start_simulation() step_simulation() pause_simulation() resume_simulation() reset_simulation() build_report()
[Heading 2] 3. 算法标准
[Heading 3] Dijkstra
[Normal (Web)] 规则：
[HTML Preformatted] 权重 = length / speed × congestionFactor normal/recovered: congestionFactor = 1 congested: congestionFactor = 1.5 closed/damaged/repairing: 不可通行
[Normal (Web)] 返回：
[HTML Preformatted] path cost distance blockedEdgesSkipped
[Heading 3] 方案生成
[Normal (Web)] 必须生成三种：
[HTML Preformatted] nearest：选择到达时间最短的 idle 队伍 skill_first：选择技能最匹配的 idle 队伍，再按到达时间排序 collaboration：选择两支较快 idle 队伍协同抢修
[Normal (Web)] 技能匹配规则：
[HTML Preformatted] road_collapse → road_repair waterlogging → drainage bridge_damage → bridge_repair pipe_burst → pipe_repair traffic_accident → general 或 road_repair
[Normal (Web)] 修复时间：
[HTML Preformatted] repairTime = workload / teamEfficiency
[Normal (Web)] 协同方案：
[HTML Preformatted] repairTime = workload / (team1Efficiency + team2Efficiency) × 0.85 cost 增加 affectedVehicles 降低
[Heading 2] 4. 状态联动标准
[Heading 3] 创建事件
[HTML Preformatted] event.status = created edge.status = closed / damaged
[Heading 3] 生成方案
[HTML Preformatted] event.status = planned plans 写入 plans.json
[Heading 3] 启动仿真
[HTML Preformatted] event.status = running edge.status = repairing selected teams.status = busy simulation.status = running
[Heading 3] 仿真完成
[HTML Preformatted] event.status = finished edge.status = recovered selected teams.status = idle teams.locationNode = 事故道路 fromNode simulation.status = finished progress = 100
[Heading 2] 5. 后端验收标准
[Normal (Web)] 后端完成后必须满足：
[HTML Preformatted] python main.py 能启动 访问首页不报错 所有接口返回 {code,message,data} CRUD 操作能真实写入 JSON 非法输入有错误提示 Dijkstra 能绕开 closed/damaged 没有 idle 队伍时方案生成失败并提示 仿真能更新事件、道路、队伍状态 报告能聚合事件、方案、仿真数据
